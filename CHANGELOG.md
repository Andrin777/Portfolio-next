# Changelog

Development history of the portfolio project. The commit history covers the
CMS migration and deployment work (18 commits, 9–15 June 2026). The iterations
below map that activity to the six development phases described in the README.

---

## Iteration 1 — Prototype
*Before repo initialisation*

Single-file HTML portfolio (`portfolio_V10.html`) with hard-coded project
content and a canvas animation (ASCII/particle effect). No build step, no
dependencies — opened directly in the browser.

**Key decisions:**
- Canvas animation written from scratch; carried forward into the Next.js version
- All nine projects and the full copy were in the HTML; later used as the source
  for the automated Sanity import script

---

## Iteration 2 — Next.js migration
*Before first tracked commit*

Scaffolded a Next.js 15 (App Router) project with TypeScript. Split the
monolithic HTML into layout, page, and component files.

**Key decisions:**
- App Router chosen over Pages Router for nested layouts and the embedded Studio route
- `src/` directory layout with `components/`, `lib/`, `sanity/` for a clean separation

---

## Iteration 3 — Sanity integration
*9 June 2026 · commits `2aa439c`, `f0d1c31`, `bbe4297`, `8d385de`*

Set up the Sanity project, defined content schemas (projects, site settings,
stack tools, tool receipts, work topics), connected the GROQ client, and wrote
the one-time import script that reads the legacy HTML and uploads all content.

**Notable fixes:**
- `@sanity/image-url` ships only a default export; removed the non-existent
  named import that caused a build error (`8d385de`)
- Adjusted `sanity.config.ts` to use the correct env-variable names (`bbe4297`)

---

## Iteration 4 — Styling & responsiveness
*9–10 June 2026 · commits `fb85305`, `b4e2e17`, `c35b197`, `b7e4dc8`, `a24ec0b`, `85e3134`*

Applied the full design system: typography (Schriftli / Inter), colour tokens,
layout grid, project cards, and detail pages. Added a dedicated mobile layout.

**Notable fixes:**
- Pinned Sanity peer-dependency versions to resolve a runtime mismatch that
  surfaced after styling work (`b7e4dc8`)
- Work-page design iterated across three commits before settling on the
  current card + detail layout

---

## Iteration 5 — Accessibility pass
*10 June 2026 · commits `b9b5f97`, `d948dc0`, `53e3f69`*

Audited and corrected semantic HTML: fixed heading hierarchy (h1→h2→h3 order),
added ARIA attributes, replaced presentational `<div>` elements with
landmark elements, and removed files that were no longer referenced.

**Key changes:**
- Heading order corrected site-wide (`d948dc0`)
- ARIA labels on icon-only buttons and navigation landmarks (`b9b5f97`)
- Deleted leftover assets and dead imports from the migration (`53e3f69`)

---

## Iteration 6 — Deployment & debugging
*10–11 June 2026 · commits `2aa439c`, `eac3a74`, `4d39e28`, `8d385de`, `688134c`, `194f742`*

Configured Netlify continuous deployment from the `main` branch via
`netlify.toml`. Resolved a series of build failures before the first clean
production build.

**Root causes resolved:**
- Repo structure had the Next.js app one level too deep for Netlify's base-
  directory detection; fixed by aligning `netlify.toml` with the actual path
  (`2aa439c`)
- `.sanity/runtime/` was auto-generated by the Studio and committed, causing
  stale-file conflicts on CI; added to `.gitignore` and removed from tracking
  (`eac3a74`)
- Sanity env vars (`NEXT_PUBLIC_SANITY_PROJECT_ID`, `NEXT_PUBLIC_SANITY_DATASET`)
  are not read from `.env.local` in production — must be set manually in
  Netlify's environment-variable dashboard
- Performance: enabled Next.js `<Image>` optimisation and removed unused CSS
  (`4d39e28`)
